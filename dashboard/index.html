🔬 Agentic AI — Final Project · April 2026

Can reflection make AI agents smarter?

This dashboard runs and tracks experiments from our Agentic AI course project at FAST NUCES. We study whether giving an LLM agent structured feedback about its own failures — before each retry — improves its performance on real machine learning tasks.

What is this research?

The benchmark: MLAgentBench is a framework where a language model acts as an autonomous ML researcher. It reads data, writes code, runs experiments, and iterates — all without human guidance — trying to beat a baseline score on standard ML tasks.

The problem: Even the best models hallucinate improvements. They often repeat the same mistakes across attempts because they have no memory of what already failed.

Our contribution — Cross-Run Reflexion: After each failed attempt, we ask the LLM to produce a structured post-mortem: what strategies were tried, why they didn't work, and what to do differently. That critique is injected into the next attempt's prompt. The agent now starts each retry with explicit memory of its own past failures.

The hypothesis: Condition C (Reflexion) should outperform Condition A (baseline) and Condition B (extra retries with no feedback), because the reflection provides a qualitative signal that raw compute alone cannot.

The Three Conditions
A
Baseline
The agent gets one attempt with no prior feedback. This replicates the original MLAgentBench paper setup. Whatever score it gets cold is the baseline performance of a vanilla LLM agent.
Controls for: raw capability
B
Multi-Attempt
The agent gets three attempts but receives no structured critique between them — just a fresh start. This isolates whether improvement in C comes from reflection specifically, or simply from having more compute and retries.
Controls for: extra compute
C
Reflexion Agent
Three attempts + a post-mortem between each. The LLM critiques its own trajectory: strategies tried, what failed, what to try next. That critique is prepended to the next attempt's system prompt.
Our contribution: reflection

How Condition C Works
1
Attempt 1 — Cold Start
The agent reads the task description and baseline code, then autonomously writes and runs ML code for up to 30 steps. At the end, the benchmark scores its final submission.
2
Post-Mortem Generation
If the score didn't beat the baseline, we send the agent's full action history to the LLM and ask it to produce a structured critique: what was tried, why it failed, what to do differently, what to avoid.
3
Reflection Injection
The post-mortem is prepended to the next attempt's system prompt. The agent now starts attempt 2 knowing exactly what its past self already tried and why it didn't work.
4
Attempt 2 (and 3) — Informed Retry
The agent retries with the critique in context. If it fails again, the cycle repeats: another post-mortem, another injection. Up to 3 attempts total.
5
Final Score
The best score across all attempts is recorded. Improvement is measured as Δ% = (baseline − score) / baseline for lower-is-better metrics, inverted for higher-is-better.
